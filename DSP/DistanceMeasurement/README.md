#  Distance Measurement (Range FFT)

This module handles the **Fast-Time** signal processing. Its goal is to locate the physical distance of targets by extracting the beat frequency ($f_b$) from the raw IF signal.

##  The Mathematics

Distance ($R$) is directly proportional to the beat frequency ($f_b$) generated by the mixer:
$$R = \frac{c \cdot T_c \cdot f_b}{2B}$$

* $c$: Speed of Light
* $T_c$: Chirp Duration
* $B$: Swept Bandwidth

### Range Resolution ($\Delta R$)
The absolute precision of this module is dictated entirely by the hardware bandwidth ($B$). Targets closer than $\Delta R$ will merge into a single FFT bin.
$$\Delta R = \frac{c}{2B}$$



##  Algorithmic Flow

1. **DC Mean Subtraction:** Raw ADC data contains a large DC bias and static TX-RX leakage. If left uncorrected, this creates a massive peak at 0 Hz (0 meters) that blinds the radar to close targets.
   $$x_{clean}[n] = x[n] - \mu_x$$
2. **Windowing:** A standard Hanning or Hamming window is applied to the time-domain data to reduce spectral leakage (sidelobes) before the FFT.
3. **The Fast Fourier Transform (FFT):** Converts the time-domain waveform into the frequency domain.
